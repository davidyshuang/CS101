# 人民银行大额反洗钱与可疑数据报送 - 知识图谱与数据血缘管理平台

本项目旨在构建一个面向人民银行大额反洗钱（AML）和可疑交易数据报送的大数据平台，主要包含以下四大核心能力：
1. 大数据平台的元数据管理
2. 数据血缘管理
3. 本体建模
4. 知识图谱表示

## 1. 总体架构设计与开源选型建议

为了快速、稳定地搭建该平台，建议采用以下主流开源软件栈进行组合：

### 1.1 元数据管理与数据血缘 (Metadata & Data Lineage)
**推荐方案：DataHub 或 Apache Atlas**
* **DataHub (由 LinkedIn 开源)**: 现代化的元数据搜索和发现平台，原生支持优秀的 UI、强大的数据血缘可视化（Data Lineage），并且易于与各种数据库、Kafka、Spark 等集成。
* **Apache Atlas**: 传统 Hadoop 生态（如 Hive、HBase、Kafka 等）中数据血缘和元数据管理的标准，与底层大数据组件结合非常紧密。
* **结论**: 如果是现代化的大数据架构，推荐使用 **DataHub** 或 **OpenMetadata**，其接口和展现层更加灵活。

### 1.2 本体建模 (Ontology Modeling)
**推荐方案：Protégé / WebProtégé + Apache Jena**
* **Protégé / WebProtégé**: 斯坦福大学开发的本体编辑工具，支持 OWL、RDF 规范，非常适合反洗钱业务专家和建模人员进行早期的领域本体概念（如：账户、客户、交易、高风险实体等）构建。
* **Apache Jena**: Java 框架，支持读取本体文件（OWL/TTL），可以作为后端解析本体规则，进行推理解析的核心组件。

### 1.3 知识图谱存储与表示 (Knowledge Graph Representation)
**推荐方案：Neo4j (社区版) / NebulaGraph + AntV G6 / ECharts**
* **图数据库层**: 
  * **Neo4j**: 目前最流行的图数据库，社区版免费，支持 Cypher 语句，生态极其完善，非常适合做反洗钱交易网络分析、洗钱环路检测。
  * **NebulaGraph / HugeGraph**: 如果是海量数据（千亿级节点/边），推荐采用国产开源分布式图数据库 NebulaGraph。
* **可视化表示层**:
  * 平台的前端展示推荐使用 **AntV G6** 或 **Cytoscape.js**。G6 对图数据的定制化渲染（如反洗钱网络中的关键节点标红、资金流向动画等）支持非常好。

### 1.4 平台前后端应用框架
* **后端 API**: Python (FastAPI / Django) 或 Java (Spring Boot)。Python 在集成图算法、数据分析处理方面有天然优势。
* **前端框架**: Vue 3 或 React，结合 Element-Plus / Ant Design。

## 2. 反洗钱业务场景映射

1. **元数据管理**: 管理人行报送规范中的字段定义（如《金融机构大额交易和可疑交易报告管理办法》中的数据字典），将数据库表字段与标准报送字段对齐。
2. **数据血缘**: 追踪一条大额交易数据从核心系统产生，经过 ETL 清洗、跑批规则引擎，最终生成 XML/JSON 报文发送给人行的完整链路，便于排查漏报和数据质量问题。
3. **本体建模**: 构建“反洗钱领域本体”，例如定义实体类（自然人、法人、账户、银行卡）、对象属性（转账、实控人、代办人）、数据属性（交易金额、风险等级）。
4. **知识图谱表示**: 将复杂的交易流水实例化为知识图谱，通过图查询（如查找 A 通过 N 个中间账户最终转账给 B 的链路）识别隐蔽的洗钱网络，直观地在前端展示给审核人员。

## 2. 层级说明 & 角色对应

1. **数据源层**
   所有原始数据来源，不改造原有业务系统，仅做**抽取同步**；是BO识别的数据底座。
2. **数据融合治理层**
   按**本体定义的语义标准**做数据对齐、清洗、脱敏，解决多源数据孤岛、口径不一致问题。
3. **知识图谱存储计算层**
   本体的**实例化载体**，用图结构存储企业、自然人、股权、控制关系；支撑**多层股权穿透**核心能力。
4. **语义规则&推理层**
   加载 WebProtégé 产出的 **OWL本体 + SWRL监管规则**，把央行受益所有人判定标准转为机器可执行逻辑。
5. **应用交互层**
   面向四类用户：合规部、业务人员、架构运维、审计师，提供可视化、操作、报表、溯源能力。
6. **本体建模平台（WebProtégé）**
   独立协作环境，**先建模、再落地**，输出领域本体与规则，向下同步到规则引擎与知识图谱。

# 二、核心数据流架构图（业务主流程）

展示**从建模→数据接入→识别→审计**完整流转，适配受益所有人全流程：

```
WebProtégé本体建模平台
        │
        ▼
  导出 OWL本体 + SWRL规则
        │
        ▼
┌─────────────────────────┐
│ 规则引擎/推理服务        │◄──────────┐
└───────────┬─────────────┘           │
            │                         │
            ▼                         │
┌─────────────────────────┐           │
│ 知识图谱(图数据库)       │◄─── 多源数据ETL/CDC
│ （实体+关系+属性）       │           内/外部数据源
└─────┬───────────┬───────┘
      │           │
      ▼           ▼
┌─────────┐   ┌───────────────┐
│BO自动识别│   │ 图谱可视化查询 │
└────┬────┘   └───────┬───────┘
     │                │
     └────────┬───────┘
              │
              ▼
    【应用工作台】
    ├─ 合规人员：核验、风险预警、监管报表
    ├─ 业务人员：客户尽调、准入查询
    ├─ 审计人员：全链路溯源、日志核查、证据导出
    └─ 管理员：权限、监控、运维
```

**流程简述**

1. 业务+技术团队在 **WebProtégé** 完成领域本体、监管规则建模；
2. 本体&规则下发至推理引擎；
3. 内外数据源经ETL治理后，按本体语义灌入知识图谱；
4. 推理引擎 + 图计算 联合执行**受益所有人穿透判定**；
5. 结果、图谱、日志统一在前端应用层呈现，服务全角色用户；
6. 所有操作、计算、数据来源**全程留痕**，支撑审计。

### 部署关键说明（贴合你现状）

1. **WebProtégé 独立集群**
   依赖 MongoDB，部署在内网，仅建模人员访问；输出本体文件向下游同步，**不直接对外暴露**。
2. **网络安全（金融合规强要求）**
   - 所有服务仅**内网网段**可访问，禁止公网；
   - MongoDB、图数据库端口仅监听内网IP，不对外；
   - 全链路HTTPS、操作日志、访问日志留存，满足审计要求。
3. **高可用设计**
   应用、图数据库、WebProtégé 均采用集群/主从架构，搭配定时备份，满足金融生产级RTO/RPO。
